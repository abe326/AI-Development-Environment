# aide-ops-status: 全タスクの対応状況サマリ

全タスクのステータスを分析し、状況サマリと改善提案を行う。

## 手順

### 1. tasks-タスク.md の読み込み

docs/00_pm-管理/tasks-タスク.md を読む。他のファイルは読まない。

### 2. サブエージェントによる並列分析

**Sub-A: ステータス別サマリ**

```
## タスク対応状況サマリ（YYYY-MM-DD時点）

### 概要
- OPEN: X件
- IN_PROGRESS: X件
- RESOLVED: X件
- CLOSED: X件
- WONTFIX: X件
- **合計: X件（未完了: X件）**

### OPENタスク（優先度順）
| ID | タイトル | 優先度 | 起票日 | 経過日数 |
|---|---|---|---|---|

### IN_PROGRESSタスク
| ID | タイトル | 優先度 | 担当 | 起票日 | 経過日数 |
|---|---|---|---|---|---|
```

**Sub-B: 滞留分析・優先度見直し提案**

以下を分析して提案する：
- **滞留案件**: 7日以上 OPEN のまま、または 14日以上 IN_PROGRESS のまま
- **優先度の見直し**: 状況変化により優先度を上げ/下げすべきタスク
- **ボトルネック**: 特定の担当者に集中していないか
- **傾向**: 最近のタスク発生パターン（同種の問題が繰り返していないか）

### 3. 結果の表示

サマリと分析結果を統合して表示する。

### 4. アクション提案

具体的な次のアクションを提案する：
- 最優先で対応すべきタスク
- 優先度の変更が必要なタスク
- WONTFIX の検討が必要な長期滞留タスク

## 注意事項

- 表示・分析のみ。ファイルの更新は行わない（更新はユーザーの指示で行う）
- 週次レビューや報告の素材として活用できる
