# aide-pm-issues: 課題ステータス確認

docs/00_pm-管理/issues-課題.md の課題一覧を分析し、ステータス別サマリと次のアクション提案を行う。

## 手順

### 1. issues-課題.md の読み込み

docs/00_pm-管理/issues-課題.md を読み込む。他の履歴ファイルは読まない。

### 2. サブエージェントによる並列分析

**Sub-A: ステータス別サマリ生成**

以下の形式でサマリを生成する：

```
## 課題サマリ（YYYY-MM-DD時点）

### OPEN（未着手）: X件
| ID | タイトル | 優先度 | 担当 | 起票日 |
|---|---|---|---|---|

### IN_PROGRESS（対応中）: X件
| ID | タイトル | 優先度 | 担当 | 起票日 |
|---|---|---|---|---|

### RESOLVED（解決済み）: X件
（直近5件のみ表示）

### CLOSED: X件 / WONTFIX: X件
```

**Sub-B: 優先度・滞留分析 + アクション提案**

- 高優先度のOPEN課題を強調
- 長期滞留（起票から7日以上OPENのまま）の課題を検出
- IN_PROGRESSが多すぎる場合に警告
- 各OPEN/IN_PROGRESS課題に対して次のアクションを提案

### 3. 結果の統合・表示

サマリと分析結果を統合してユーザーに表示する。

## 追加機能

ユーザーの指示に応じて以下も実行する：

- **課題の追加**: 新しい課題を issues-課題.md に起票
- **ステータス更新**: 指定課題のステータスを変更
- **アーカイブ**: CLOSED/WONTFIX が20件を超えた場合、issues-archive.md への移動を提案

## 注意事項

- issues-課題.md のみを読む（コンテキスト最小読み込み）
- 詳細な検討履歴が必要な場合は、ユーザーの指示を待ってからリンク先を読む
