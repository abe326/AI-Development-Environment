# aide-dev-migrate: 依存移行・アップデート

依存ライブラリやフレームワークのバージョンアップ・移行を安全に行う。CVE脆弱性チェックを含む。

## ユースケース

- 依存ライブラリのセキュリティアップデート（CVE対応）
- メジャーバージョンアップ（破壊的変更あり）
- フレームワーク移行（例: Express → Fastify, CRA → Vite）
- ランタイムアップデート（例: Node.js 18 → 20）

## 手順

### 1. 移行対象の確認

ユーザーに以下を確認する（引数で指定されている場合はスキップ）：

- **移行種別**: `security`（CVE対応） / `update`（バージョンアップ） / `migrate`（フレームワーク移行）
- **対象**: パッケージ名 / フレームワーク名 / `all`（全依存の棚卸し）

### 2. 事前読み込み

1. パッケージ管理ファイルを読む:
   - `package.json` / `package-lock.json`（Node.js）
   - `requirements.txt` / `pyproject.toml` / `Pipfile.lock`（Python）
   - `go.mod`（Go）
   - `Cargo.toml`（Rust）
   - その他、プロジェクトの技術スタックに応じたファイル
2. `docs/00_pm-管理/技術スタック選定.md` を読む
3. 対象ライブラリを使用しているソースコード（`src/` 配下）を特定する

### 3. サブエージェントによる並列分析

#### Sub-A: CVE脆弱性スキャン

パッケージマネージャのaudit機能とCVEデータベースを照合する。

```bash
# Node.js
npm audit --json 2>&1

# Python
pip-audit --format=json 2>&1

# Go
govulncheck ./... 2>&1

# 汎用（OSS）
trivy fs --format json . 2>&1
```

ツールが未導入の場合は、AIがパッケージバージョンと既知のCVEを照合する（WebSearch でNVD/GitHub Advisory Databaseを参照）。

**CVEレポート:**

| パッケージ | 現バージョン | CVE | 重大度 | 修正バージョン | 影響範囲 |
|---|---|---|---|---|---|
| lodash | 4.17.20 | CVE-2021-23337 | High | 4.17.21 | src/utils/ |

#### Sub-B: 破壊的変更の分析

アップデート対象のCHANGELOG / Migration Guide を確認し、破壊的変更を列挙する。

| 変更 | 影響するファイル | 対応方法 |
|---|---|---|
| APIの引数変更 | src/auth/login.ts | 引数を新形式に更新 |
| 非推奨APIの削除 | src/utils/helper.ts | 代替APIに置換 |

#### Sub-C: 影響範囲の特定

対象ライブラリを import/require しているファイルを全て特定する。

```bash
# import文を検索
grep -rn "from '[対象パッケージ]'" src/
grep -rn "require('[対象パッケージ]')" src/
```

#### Sub-D: テスト影響分析

移行によって影響を受けるテストを特定する。

### 4. 移行計画の生成

分析結果を統合し、移行計画を生成する。

出力先: `docs/00_pm-管理/migration-YYYYMMDD-{対象}.md`

```markdown
# 移行計画: {対象パッケージ/フレームワーク}

## 基本情報
- 作成日: YYYY-MM-DD
- 移行種別: [security / update / migrate]
- 現バージョン: X.X.X
- 移行先バージョン: Y.Y.Y

## CVE脆弱性サマリ

| 重大度 | 件数 |
|---|---|
| Critical | X |
| High | X |
| Medium | X |
| Low | X |

### 対応必須CVE

| CVE | パッケージ | 重大度 | 修正バージョン | 影響 |
|---|---|---|---|---|
| CVE-XXXX-XXXX | xxx | Critical | X.Y.Z | [影響の説明] |

## 影響範囲

| 影響するファイル | 変更内容 | リスク |
|---|---|---|
| src/auth/login.ts | API引数の変更 | 低 |
| src/utils/helper.ts | 非推奨API置換 | 中 |
| tests/auth.test.ts | テスト修正 | 低 |

## 移行手順（段階的）

### Step 1: セキュリティ修正（パッチバージョン）
- [ ] `npm audit fix` / `pip-audit --fix` で自動修正可能なものを適用
- [ ] テスト実行で既存機能に影響がないことを確認

### Step 2: マイナーバージョンアップ
- [ ] 非推奨警告の確認と対応
- [ ] 対応するコード変更
- [ ] テスト実行

### Step 3: メジャーバージョンアップ（該当時）
- [ ] 破壊的変更への対応
- [ ] 対応するコード変更
- [ ] 全テスト実行

## リスクと対策

| リスク | 発生確率 | 影響度 | 対策 |
|---|---|---|---|
| 互換性の問題 | 中 | 高 | 段階的に移行、各ステップでテスト |
| パフォーマンス劣化 | 低 | 中 | ベンチマーク比較 |

## ロールバック手順
1. `git revert` で移行コミットを巻き戻す
2. `npm ci` / `pip install -r requirements.txt` で依存を復元
```

### 5. ユーザー承認後の移行実行

移行計画の承認を得た後、段階的に実行する：

1. **ブランチ作成**: `migrate/{対象}-{バージョン}`
2. **依存更新**: パッケージマネージャで更新
3. **コード修正**: 破壊的変更への対応（`/aide-dev-code` と連携）
4. **テスト実行**: `/aide-dev-test` で全テスト実行
5. **CVE再スキャン**: 移行後に脆弱性が解消されたか確認

### 6. 結果の記録

- `docs/00_pm-管理/issues-課題.md` に移行結果を記録
- `docs/00_pm-管理/技術スタック選定.md` のバージョン情報を更新

## 定期実行の推奨

以下のタイミングで定期的に `aide-dev-migrate security` を実行することを推奨：

- スプリント開始時
- リリース前
- CVEアラート受信時

## 前提条件

| ツール | 用途 | 必須/任意 |
|---|---|---|
| npm audit / pip-audit / govulncheck | CVEスキャン | いずれか必須 |
| trivy | 汎用脆弱性スキャン | 任意 |

ツールが未導入の場合、AIがWebSearchでCVEデータベースを参照してチェックする。

## 注意事項

- Critical/High のCVEは即時対応を推奨
- メジャーバージョンアップは影響範囲が大きいため、必ず段階的に行う
- 移行は必ずブランチで行い、mainに直接pushしない
- ロールバック手順を事前に確認してから移行を開始する
