# aide-pm-diagram: 図の作成

成果物やブレストファイルの内容から .drawio.svg 形式の図を生成する。

## 手順

### 1. 対象の特定

ユーザーに以下を確認する（引数で指定されている場合はスキップ）：
- 対象のファイルパス（成果物、ブレストファイル等）
- 図の種類（未指定の場合は内容から判断）：
  - フロー図（業務フロー、処理フロー）
  - 構成図（アーキテクチャ、システム構成）
  - ER図（データモデル）
  - シーケンス図（API連携、処理順序）
  - ユースケース図
  - その他

### 2. 対象ファイルの読み込み

- 指定されたファイルを読む
- 他の履歴ファイルは読まない

### 3. .drawio.svg の生成

draw.io の XML メタデータを埋め込んだ SVG ファイルを生成する。

**ファイル形式のポイント：**
- draw.io で再編集可能な XML メタデータを含む
- GitHub 上では SVG として表示される

**図の作成ルール：**
- 要素は簡潔に（1要素に長文を入れない）
- 色分けで視認性を確保（ステータス、分類など）
- 矢印・線でフローや関係を明示
- 凡例を含める（色の意味など）

### 4. 元のファイルへの参照追記

元のファイルに以下の形式で図の参照を追記する：

```markdown
## 図
![図の説明](ファイル名.drawio.svg)
```

### 5. 完了報告

生成した図のファイルパスと概要を報告する。
「draw.io で開いて修正・仕上げが可能です」と案内する。

## 出力先

対象ファイルと同じディレクトリに、対応する名前で配置する。
例: `docs/02_design/設計書-ユーザー登録.drawio.svg`

## 注意事項

- 図がなくても成果物だけで成立する（図は必須ではない）
- 図だけの単独ファイルも可（人間が作成する場合）
- 提出用には draw.io から PNG/PDF にエクスポートすることを案内する
