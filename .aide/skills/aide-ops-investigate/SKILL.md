# aide-ops-investigate: タスクの調査・原因分析

指定されたタスクIDの調査を実施し、原因候補を特定する。

## 手順

### 1. タスク情報の確認

- 引数からタスクID（T-XXX）を取得する（未指定の場合はユーザーに確認）
- docs/00_pm-管理/tasks-タスク.md から該当タスクの情報を読む
- docs/tasks-タスク/T-XXX-*/ 内の対応記録ファイル（YYYYMMDD-*.md）を読む
- タスクのステータスを IN_PROGRESS に更新する

### 2. サブエージェントによる並列調査

**Sub-A: ソースコード分析**
- 問い合わせ内容に関連するソースコードを特定
- 問題が発生しうる箇所を分析
- 原因候補を列挙

**Sub-B: ログ・エラー情報の分析**
- ユーザーから提供されたログやエラー情報を分析
- エラーの発生条件・再現手順を整理
- 関連する設定やデータの確認

**Sub-C: 影響範囲の特定**
- 問題箇所に依存するコード・機能を特定
- 修正した場合の影響範囲を分析
- 関連するテストを確認

### 3. 結果の統合・報告

原因候補を確度順に整理してユーザーに報告する：

```
## 調査結果: T-XXX

### 原因候補（確度順）
1. **[候補1]**（確度: 高）
   - 根拠: [分析結果]
   - 該当箇所: [ファイル名:行番号]

2. **[候補2]**（確度: 中）
   - 根拠: [分析結果]
   - 該当箇所: [ファイル名:行番号]

### 影響範囲
- [影響を受ける機能・ファイルの一覧]

### 推奨アクション
- [次に行うべきこと]
```

### 4. 記録の更新

**tasks-タスク.md のステータスを IN_PROGRESS に更新。**

**対応記録ファイル（docs/tasks-タスク/T-XXX-*/YYYYMMDD-*.md）の対応履歴に追記：**
```markdown
### YYYY-MM-DD 調査

**原因候補（確度順）:**
1. [候補1]（確度: 高） — [根拠・該当箇所]
2. [候補2]（確度: 中） — [根拠・該当箇所]

**影響範囲:** [影響を受ける機能・ファイル]

**推奨アクション:** [次に行うべきこと]
```

また、ファイルの基本情報セクションのステータスを「調査中」に更新する。

### 5. 次のステップの案内

- 原因が特定できた場合 → /aide-ops-fix で修正対応を案内
- 追加調査が必要な場合 → 必要な情報をユーザーに依頼

## 注意事項

- 過去の対応記録（docs/tasks-タスク/）は、フォルダ名一覧から関連しそうな1-2件のみ読む
- 推測だけで原因を断定しない。根拠を明示する
