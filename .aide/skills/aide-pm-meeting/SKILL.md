# aide-pm-meeting: 議事録作成 + 課題起票

打合せメモを構造化された議事録にし、未決事項を課題として自動起票する。

## 手順

### 1. 入力の確認

ユーザーから以下を受け取る：
- 打合せメモ（箇条書き、音声書き起こし、自由形式いずれも可）
- 日付（未指定の場合は今日の日付を使用）
- 議題名（未指定の場合はメモ内容から推定）

### 2. サブエージェントによる並列処理

以下を並列で実行する：

**Sub-A: 議事録の構造化**
打合せメモを以下のテンプレートに沿って構造化する：

```markdown
# [議題]

## 基本情報
- 日付: YYYY-MM-DD
- 参加者: [メモから抽出]
- 目的: [メモから推定]

## 議題と決定事項
1. [議題] → [決定事項]

## 未決事項・課題
- [課題内容] → issues-課題.md に起票

## アクションアイテム
- [ ] [担当者] [期限] [内容]

## 次回予定
[あれば記載]
```

**Sub-B: 課題・未決事項の抽出**
メモから未決事項・課題を抽出し、issues-課題.md への追加案を作成する。各課題には：
- 課題IDの発番（既存の最大IDの次番号）
- 優先度の判定（高/中/低）
- 起票元の記録（議事録ファイル名）

**Sub-C: アクションアイテムの抽出・期限整理**
担当者・期限・内容を整理する。

### 3. 結果の統合・出力

- 議事録ファイルを、テーマに最も関連するフェーズの meeting-notes-打合せ/ に出力する
  - 要件定義の打合せ → `docs/01_requirements/meeting-notes-打合せ/YYYYMMDD-議題.md`
  - 設計の打合せ → `docs/02_design-設計/meeting-notes-打合せ/YYYYMMDD-議題.md`
  - 製造計画の打合せ → `docs/03_plans-製造計画/meeting-notes-打合せ/YYYYMMDD-議題.md`
  - フェーズが不明な場合はユーザーに確認する
- docs/00_pm-管理/issues-課題.md に新規課題を追加（ステータス: OPEN）
- 既存課題に関連する議論があれば検討履歴に追記

### 4. ユーザーへの報告

出力した議事録のサマリと、起票・更新した課題の一覧を表示する。

## コンテキスト最小読み込みルール

- docs/00_pm-管理/issues-課題.md を読む（課題IDの発番と既存課題の確認のため）
- 他の議事録やブレスト履歴は読まない
