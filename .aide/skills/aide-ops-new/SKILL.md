# aide-ops-new: 新規問い合わせ受付 + タスク起票

新規の問い合わせ・課題を受付し、タスクとして起票する。

## 手順

### 1. 問い合わせ内容の受付

ユーザーから以下を受け取る：
- 問い合わせ内容（不具合報告、要望、質問など）
- 報告者（わかれば）
- 緊急度（わかれば）

### 2. サブエージェントによる並列処理

**Sub-A: 過去の類似タスクを検索**
- docs/00_pm-管理/tasks-タスク.md から類似のタスクを検索（キーワードマッチ）
- docs/tasks-タスク/ のフォルダ名一覧から関連しそうなものを特定（フォルダ名のみ、中身は読まない）
- 類似タスクがあれば報告

**Sub-B: タスク情報の構造化 + T-XXX発番**
- 問い合わせ内容を構造化
- 既存の最大IDの次番号を発番
- タスクテンプレートに沿って情報を整理

**Sub-C: 優先度の判定提案**
- 影響範囲、緊急度、報告内容から優先度（高/中/低）を判定
- 判定理由を提示

### 3. 起票内容のユーザー確認

以下をユーザーに提示し、確認を得る：
- タスクID（T-XXX）
- タイトル
- 優先度と判定理由
- 類似の過去タスク（あれば）

### 4. tasks-タスク.md への起票

承認後、docs/00_pm-管理/tasks-タスク.md のタスク一覧テーブルに行を追加する：
```markdown
| T-XXX | [タイトル] | [優先度] | OPEN | YYYY-MM-DD | — | 問い合わせ受付 |
```

### 5. タスクフォルダ・対応記録ファイルの作成

docs/tasks-タスク/ 配下に専用フォルダとファイルを作成する：

**フォルダ構成:**
```
T-XXX-[タイトル短縮]/
├── YYYYMMDD-[タイトル短縮].md   ← 対応記録（Single Source of Truth）
└── slides-資料/                  ← 報告資料用（空で作成）
```

**対応記録ファイルの内容：**
```markdown
# T-XXX: [タイトル]

## 基本情報

- **タスクID**: T-XXX
- **受付日**: YYYY-MM-DD
- **報告者**: [報告者]
- **種別**: [不具合/要望/質問]
- **優先度**: [高/中/低]
- **ステータス**: 受付

## 問い合わせ内容

[ユーザーから受け取った問い合わせ内容をそのまま記録]

## 対応履歴

### YYYY-MM-DD 受付

問い合わせを受付し、T-XXX として起票。
[類似タスクの有無、初期所見があれば記載]
```

### 6. 完了報告

起票したタスクのサマリと、次のアクション（/aide-ops-investigate で調査開始）を案内する。

## 規模判断

以下に該当する場合、新規開発フロー（大規模）への切り替えを提案する：
- 変更ファイル数が10を超える見込み
- 工数が3人日を超える見込み
- 複数機能・サービスに影響する
- アーキテクチャ変更が必要
