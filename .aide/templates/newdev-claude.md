## フェーズ別AIの利用方針

| Phase | AIの役割 | 人間の役割 |
|---|---|---|
| 0 企画 | 要件整理、ユーザーストーリー生成 | 判断・承認 |
| 1 設計 | アーキテクチャ案生成、仕様書・図ドラフト | 技術選定・レビュー |
| 2 計画 | タスク分解、依存関係分析 | 優先順位決定 |
| 3 実装 | 仕様に基づくコード生成 | 仕様準拠レビュー |
| 4 テスト・QA | テスト仕様書作成(UT/IT)、テスト実行・証跡収集、品質レポート生成 | QA判断 |
| 5 リリース | リリースノート生成、手順書作成 | 最終承認 |

## 成果物一覧

### 管理資料（フェーズ横断）

`docs/00_pm-管理/` で管理。フェーズを通じて常時更新する。

| 成果物 | ファイル | 備考 |
|---|---|---|
| 課題管理 | issues-課題.md | 全フェーズ共通の課題追跡 |
| スケジュール | schedule-スケジュール.md | マイルストーン・タスク分解 |
| 見積もり | estimate-見積もり.md | フェーズ進行で精度向上（±50%→±15%） |
| 技術スタック選定 | 技術スタック選定.md | 選定理由・比較結果 |
| スライド | slides-資料/ | 説明・報告用（Marp形式） |

### フェーズ別成果物

各フェーズの成果物は `deliverables-成果物一覧.md` で定義されています。プロジェクトに合わせてカスタマイズしてください。

| フェーズ | 成果物一覧 |
|---|---|
| 要件定義 | docs/01_requirements/deliverables-成果物一覧.md |
| 設計 | docs/02_design/deliverables-成果物一覧.md |
| 製造計画 | docs/03_plans/deliverables-成果物一覧.md |
| テスト | docs/04_testing/deliverables-成果物一覧.md |

成果物の提出用変換は `/aide-pm-export` で実行できます。
